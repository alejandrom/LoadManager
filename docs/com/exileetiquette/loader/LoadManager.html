Property detail
baseURLproperty
baseURL:String  [read-write]

The base URL to load all files from. For example, if baseURL was set to "myApp/data/", all files added to the queue (and XML file lists loaded using loadQueueFromXML()) would have "myApp/data/" prepended to their URLs, making the final URL "myApp/data/myXMLFile.xml".

dispatchProgressEventsproperty 
dispatchProgressEvents:Boolean  [read-write]

Determines if progress events are dispatched when files are loading. Set this value to true if you want to monitor the load progress of individual files. Progress events are dispatched as com.exileetiquette.loader.events.LoadManagerProgressEvent events.

loadingproperty 
loading:Boolean  [read-only]

Determines if the LoadManager instance is currently set to load files in the queue. This value is true by default, meaning files added to the queue will begin loading immediately. It will also be set to true when loadQueue() or loadQueueFromXML() is called, and set to false only when pauseQueue() is called.

public function addToQueue(url:String, priority:uint = 0, id:String = null, type:String = null, context:* = null, data:* = null):String

Adds a resource URL to the queue for loading.

Parameters
url:String — The url of the resource
 
priority:uint (default = 0) — [optional] The priority of the resource. Higher prorities will be added to the front of the queue.
 
id:String (default = null) — [optional] A unique identifier for the resource (defaults to '_fileN', where N is an incrementing number beginning at 0)
 
type:String (default = null) — [optional] The type of resource
 
context:* (default = null) — [optional] A load context object to use when loading the file
 
data:* (default = null) — [optional] Any custom data you want to store along with the file. This can be retrieved later using getFileData()

Returns
String — The id of the file that was added to the queue
addToQueueFromXML()method 
public function addToQueueFromXML(xml:XML):void

Adds files to the queue from an XML resource list. Each file node within the XML document should be formatted as follows:
<file id="fileId" type="typeId"><![CDATA[urlToFile]]></file>
Once the file has loaded, you can use getFileData() to retrieve the XML node for the file.

Parameters
xml:XML — The XML document containing a list of resources to load

Dispatched when a file has completed loading

The LoadManagerEvent.FILE_COMPLETE constant defines the value of the type property of the event object for a loadManagerFileComplete event.

The properties of the event object have the following values:

PropertyValue

cancelablefalse; there is no default behavior to cancel.
currentTargetThe object that is actively processing the LoadManagerEvent object with an event listener.
targetThe LoadManager instance that has completed loading the file.
fileIdThe id of the file that has completed loading.
dataAny custom data that was specified when the file was added to the queue.
